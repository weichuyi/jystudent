# 模块与页面讲解

本文档用于向组员说明项目各模块包含的页面与功能，便于答辩或分工。

## 概览

- 运行入口：`app.py`（主），`main.py` / `run.py`（备选）
- 核心模块位于 `app/`：包含 `forms/`, `models/`, `routes/`, `services/`, `utils/` 等子模块
- 页面模板位于 `templates/`，静态资源在 `static/`

## 代码模块说明

- `app/forms/`：表单定义（使用 Flask-WTF/WTForms），例如学生/教师/课程表单，负责表单校验规则。
- `app/models/`：包含数据模型与数据库交互逻辑（`models.py`, `base.py`），定义学生、教师、班级、课程、选课、成绩、用户等表结构。
- `app/routes/`：路由与视图函数：
  - `auth.py`：登录、登出、忘记密码、Key 重置等认证相关页面。
  - `dashboard.py`：系统首页 / 仪表盘。
  - `admin.py`：管理者后台页面路由（用户管理、备份、日志、系统信息等）。
- `app/services/`：核心业务逻辑封装，供路由调用：
  - `user_service.py`：用户账号、权限相关操作。
  - `student_service.py`：学生的增删改查与同步账号逻辑。
  - `course_service.py`：课程、选课、容量校验相关逻辑。
  - `data_service.py`：导出/导入/备份相关功能。
- `app/utils/`：工具函数与公共组件（错误处理、日志封装、字段校验等）。

## 页面与模板对应关系（主要页面）

- `templates/login.html`：登录页，由 `auth.py` 提供。
- `templates/dashboard.html`：系统首页/仪表盘，由 `dashboard.py` 提供。
- `templates/students/list.html`、`templates/students/form.html`：学生列表与新增/编辑页面，路由在 `admin.py` / 学生管理入口。
- `templates/teachers/list.html`、`templates/teachers/form.html`：教师管理页面。
- `templates/classes/list.html`、`templates/classes/form.html`：班级管理页面。
- `templates/courses/list.html`、`templates/courses/form.html`：课程管理页面与批量导入功能。
- `templates/enrollments/list.html`、`templates/enrollments/admin_form.html`：选课记录页与管理员代学生选课页面。
- `templates/scores/list.html`、`templates/scores/form.html`、`templates/scores/manual_form.html`：成绩查看与成绩手动录入页面。
- `templates/admin/*`：管理中心相关页面（备份、日志、用户管理、系统信息、数据导出等）。

## 功能流程（示例）

- 学生新增：管理员在学生表单提交 -> `student_service` 创建学生记录 -> 自动创建用户账号（同步在 `user_service`）
- 选课：学生点击选课 -> `course_service` 校验容量与重复 -> 写入选课记录 -> 若无选课记录可在录成绩时自动补建
- 成绩录入：教师/管理员在成绩页面提交 -> `services` 检查权限并写分数 -> 更新选课/成绩表

## 小组分工建议（答辩/演示）

- 前端/模板：负责 `templates/` 和 `static/` 的页面调整与样式
- 后端路由：负责 `app/routes/` 中各模块 API 与页面逻辑
- 服务层：负责 `app/services/` 的业务实现与测试
- 数据与模型：负责 `app/models/` 的表结构与迁移（备份/恢复）
- 打包与部署：负责 `build_exe.py`、release 输出与 VPS 更新脚本

## 备注

- 本项目已兼顾开发便利性（`app.py`）与模块化设计（`app/`），答辩时可先演示 `app.py` 的完整流程，再指出如何将功能回迁至 `app/` 结构以便扩展。
